# Governed Membranes GM.19 decision-chain playground evidence

Status: release-hardening product-surface overlay on exact integrated base
`28a79deae0779d607693086cd4d655a72084c3e9`.

## Scope

GM.19 completes the bounded G4 decision-chain presentation surface for the
frozen Workspace v0 facade. It adds:

- a presentation-only OCaml projection with schema
  `jacquard-governance-decision-chain-v1`;
- six deterministic backend-generated illustrative fixtures;
- a local, offline, read-only React and TypeScript viewer;
- strict client shape and closed-state validation;
- accessible evidence presentation and three-engine browser evidence; and
- CI that retains a static build artifact without deploying it.

It changes no kernel form, surface syntax, HASH_V0 rule, Governance v0 or Audit
v2 carrier, run/reconciliation bundle, policy result, authority derivation,
runtime grant, approval path, action path, or 0.1 package.

## Projection boundary

`Governance_decision_chain.of_explain` accepts the existing typed
`Governance_explain.report`. Before projecting it, the adapter rechecks
canonical Call, Proposal, BoundPolicy, assessment, Decision, Audit ordering,
authority, identity, and action-state relationships. Failure is a stable
`E1542` diagnostic. It does not evaluate policy or derive authority anew.

The projection emits six ordered typed stages:

```text
request -> assessment -> verdict -> consent -> activity -> outcome
```

Every committed artifact carries a backend-supplied full identity and opaque
subject. Resource evidence omits raw scope content and remains explicitly
separate from effect-row authority. The browser accepts only the normalized
schema. It does not accept `.jqd`, canonical forms, run bundles, or
reconciliation bundles.

The closed verdict vocabulary is `Allow`, `Ask`, `Block`, and `Simulate`.
Consent is `Not required`, `Approved`, `Denied`, `Escalated`, `Stale`, or
`Missing`. The adapter rejects a verified report that conflates non-`Ask`
verdicts with committed consent. The four released live-attempt states remain
unchanged.

## Fixture provenance

The six checked-in JSON files are generated by typed OCaml constructors:

1. `allowed.json`: `Allow`, consent not required, no action;
2. `blocked.json`: `Block`, consent not required, no action;
3. `stale-approval.json`: `Ask`, stale consent, evaluation evidence but no
   committed consent or action;
4. `transformed.json`: distinct parent and child Call identities, configured
   resource evidence, and proposal-bound approval;
5. `attempt-missing-completion.json`: proposal-bound approval, committed
   attempt and receipt digest, and a missing completion; and
6. `dry-simulation.json`: `Simulate`, consent not required, pure simulation,
   and an explicit simulation-is-not-consent marker.

Each fixture has `source: "fixture"` and `illustrative: true`. The UI displays
**Illustrative fixture — not verified evidence**. Fixture bytes are
deterministic and the compiled suite regenerates and byte-compares every file.
They exercise presentation states without claiming that a verifier observed
those events.

## Client and accessibility evidence

The package under `playground/governance/` pins Node 24.18.0 LTS, pnpm, every
dependency version, and the complete lockfile. Vite produces a static build.
Development and preview servers bind to `127.0.0.1`. There is no telemetry,
analytics, cookie, service worker, remote font, CDN, outbound fetch, or browser
persistence.

The validator rejects unknown versions, profiles, keys, artifact kinds,
states, malformed hashes, provenance mismatch, changed evidence limits,
inconsistent verdict/consent/action relationships (including attempted action
for any decision other than an approved `Ask`), explicit fields outside the
schema, and documents over 256 KiB. It validates presentation structure only;
it never authenticates the document-supplied source marker, detects secret
content inside allowed text, evaluates policy, calculates hashes, derives
authority, joins artifacts, or upgrades evidence.

The renderer uses an ordered semantic list with a heading and focus target for
each stage. Status is textual and not color-only. Full hashes are inspectable
and copyable. Missing artifacts say **No committed artifact**. Exact labels
keep type authority, resource configuration, simulation, consent, attempt,
receipt, completion, reconciliation, transformed identity, and missing
completion claims distinct.

Vitest and Testing Library cover all fixtures, snapshots, axe checks, hostile
text escaping, semantic non-confusion, input bounds, reset/rejection focus, and
keyboard order. Playwright runs Chromium, Firefox, and WebKit and covers
the exact static build, keyboard traversal, visible focus, reduced motion,
forced colors, dark-preference contrast, and absence of non-loopback
application requests.

## Honest product boundary

This is a source-checkout research review surface, not a hosted product,
operator console, approval UI, authentication system, or production security
boundary. It cannot grant consent, execute an action, retry, roll back, append
Audit, authenticate an identity, establish simulator fidelity, prove
execution, prove receipt truth, prove rollback, or establish provider truth.
Its static CI artifact is retained for review and is not deployed.

## Reproduction

```sh
eval "$(opam env)"
mkdir -p "$PWD/.scratch/tmp"
export TMPDIR="$PWD/.scratch/tmp"

opam exec -- dune build --root "$PWD" @all
opam exec -- dune runtest --root "$PWD"

JACQUARD_GOVERNANCE_PLAYGROUND_FIXTURES_OUT="$PWD/playground/governance/fixtures/generated" \
  opam exec -- dune exec --root "$PWD" test/gen_governance_playground_fixtures.exe
git diff --exit-code -- playground/governance/fixtures/generated

(
  cd playground/governance
  corepack enable
  pnpm install --frozen-lockfile
  pnpm run lint
  pnpm run typecheck
  pnpm run test
  pnpm run build
  pnpm exec playwright install chromium firefox webkit
  pnpm run test:e2e
)

sha256sum -c docs/release/governed-membranes/GM19-MANIFEST.sha256
```
